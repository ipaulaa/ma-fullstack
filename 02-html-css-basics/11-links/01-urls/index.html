URL Structure

href paths

Current URL: https://example.com/products/books/view.html

  • Absolute URL
    • href: https://google.com
    • Goes to: https://google.com
      An external website, full URL
  • Protocol-relative URL
    • href: //cdn.example.com/lib.js
    • Goes to: https://cdn.example.com/lib.js
      Uses the current page's protocol
  • Root-relative path
    • href: /about.html
    • Goes to: https://example.com/about.html
      Starts from the root of the site
  • Relative path (same folder)
    • href: details.html
    • Goes to: https://example.com/products/books/details.html
      File in the same directory as the current page
  • Relative path (current directory)
    • href: ./info.html
    • Goes to: https://example.com/products/books/info.html
      Same as above â€” current folder
  • Relative path (one folder up)
    • href: ../index.html
    • Goes to: https://example.com/products/index.html
      Moves up one directory
  • Relative path (two folders up)
    • href: ../../contact.html
    • Goes to: https://example.com/contact.html
      Moves up two directories to root
  • Query string only
    • href: ?sort=price&category=books
    • Goes to: https://example.com/products/books/view.html?sort=price&category=books
      Adds parameters to the current page
  • Anchor link (fragment identifier)
    • href: #reviews
    • Goes to: Scrolls to element with id="reviews"
      Stays on the same page
